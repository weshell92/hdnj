# 《黄帝内经》网站建设计划 plan.md

## 一、项目定位

本项目不是普通古文阅读站，也不是碎片化养生文章站，而是一个围绕《黄帝内经》构建的“中医经典知识系统网站”。

核心目标：

```text
让普通人读得懂，让学习者理得清，让研究者查得快。
```

## 二、设计原则

1. 尊重原典：知识点尽量回到原文出处、篇章、关键词与相关概念。
2. 系统呈现：把天人相应、阴阳五行、脏腑经络、气血津液、病因病机、诊法治则、四时养生连成整体。
3. 古今有别：中医古典概念不简单等同现代解剖和现代医学概念。
4. 不做诊断：本站用于学习，不替代医生，不开方，不根据症状判断疾病。

## 三、网站结构

```text
首页
├── 原典阅读
│   ├── 素问
│   └── 灵枢
├── 知识体系
│   ├── 阴阳五行
│   ├── 脏腑系统
│   ├── 经络系统
│   ├── 气血津液
│   ├── 病因病机
│   └── 治则治法
├── 四季养生
├── 名句精读
├── 学习路径
├── 古今对照
├── AI 读书助手
└── 搜索
```

## 四、八个阶段交付

### 第一阶段：项目基础搭建

完成 Next.js + TypeScript + Tailwind CSS 项目结构、全局布局、导航、首页和基础视觉风格。

### 第二阶段：原典阅读模块

完成《素问》《灵枢》列表页和篇章详情页，支持原文、断句、白话、关键词、思想解读、前后篇跳转。

### 第三阶段：知识体系模块

完成知识体系首页、概念详情页、概念关系展示和知识地图组件。

### 第四阶段：脏腑与经络模块

完成五脏六腑页面、十二经络页面、经络图谱可视化和详情说明。

### 第五阶段：四季养生与名句精读

完成四季养生页面、季节详情页、名句列表页、名句详情页，并与原文和概念关联。

### 第六阶段：搜索与学习路径

完成全站搜索、结果分类、学习路径、古今对照页面。

### 第七阶段：AI 读书助手

完成本地规则版 AI 读书助手：支持经典学习问答，拒绝诊断、开方、治疗类问题。

### 第八阶段：优化与上线

完成 SEO、响应式、免责声明、构建脚本、部署配置和项目文档。

## 五、技术栈

- Next.js App Router
- React
- TypeScript
- Tailwind CSS
- 本地 TypeScript 数据文件
- 纯前端搜索与规则版 AI 助手

## 六、安全边界

所有健康相关页面均展示：

> 本网站内容用于传统文化学习与经典阅读，不构成医疗诊断或治疗建议。如有身体不适，请咨询正规医疗机构或专业医生。
